# Skill: Code Review

Review code and suggest improvements for quality, performance, and security.

## When to use
- The user shares code and asks for feedback
- The user asks "review this", "check my code", "improve this"
- Before merging or deploying code

## Instructions
1. Read the code carefully and understand its purpose
2. Check for: bugs, security issues, performance problems, style inconsistencies
3. Suggest specific improvements with examples when possible
4. Be constructive — explain WHY each suggestion matters
5. Prioritize: critical issues > improvements > style suggestions
